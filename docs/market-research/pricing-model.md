# 과금 모델 설계 — SaaS 가격 전략

> 작성일: 2026-04-03 | 작성자: Self-Improving Agent (자동 생성)

---

## 1. 목적

Self-Improving Agent의 수익 모델을 결정한다. 타깃 사용자(1인 개발자/인디 해커)의 지불 의향과 경쟁 환경을 기반으로 최적의 과금 방식과 가격대를 선택한다.

---

## 2. 과금 방식 비교

### 옵션 A: SaaS 월정액 (Monthly Subscription)

**구조**: 월 고정 요금, 티어별 기능/레포 수 차등

**장점**
- 예측 가능한 반복 매출 (MRR)
- 사용자도 예산 예측 용이 — 인디 해커 선호 패턴
- 업셀 경로 명확 (Free → Pro → Team)
- 인디 해커 도구 시장의 표준 모델 (Vercel, Railway, Supabase 등)

**단점**
- 초기 가치 입증 전 이탈률 높을 수 있음
- 레포가 적은 사용자에게 과금 부담 느낄 수 있음

**적합도**: ⭐⭐⭐⭐⭐

### 옵션 B: 레포당 과금 (Per-Repository Pricing)

**구조**: 연결된 레포 수에 비례해 과금

**장점**
- 사용량 비례 → 공정하다는 인식
- 소규모 사용자 진입 장벽 낮음

**단점**
- 사용자가 레포 연결을 꺼릴 수 있음 (확장 억제)
- MRR 변동성 높음
- 인디 해커 생태계에서 드문 모델 → 혼란 유발 가능

**적합도**: ⭐⭐⭐

### 옵션 C: 마켓플레이스 판매 (GitHub Marketplace)

**구조**: GitHub Marketplace에서 앱으로 판매, GitHub이 결제 처리

**장점**
- 신뢰도 높음 (GitHub 플랫폼 내 구매)
- 마케팅 채널로 활용 가능 (검색 노출)
- 결제 인프라 구축 불필요

**단점**
- GitHub 수수료 (일반적으로 매출의 25%)
- 가격 정책 제한 (GitHub 규정 준수 필요)
- 비 GitHub 사용자 접근 불가 (GitLab, Bitbucket 제외)
- 마켓플레이스 경쟁 심화 중

**적합도**: ⭐⭐

---

## 3. 결정: SaaS 월정액 모델 선택

**선택 이유**:

1. **타깃 사용자 친화적**: 1인 개발자/인디 해커는 월정액 SaaS에 익숙 (Vercel $20/월, Railway $5/월, Supabase $25/월 등 이미 여러 도구를 월정액으로 사용 중)
2. **예측 가능한 MRR**: 초기 스타트업에게 가장 중요한 지표인 월 반복 매출을 안정적으로 확보
3. **업셀 경로 자연스러움**: Free → Pro → Team으로 사용자 성장에 맞는 자연스러운 확장
4. **수수료 최소화**: GitHub Marketplace 25% 수수료 대신 Stripe 2.9%만 부담

## 4. 가격 티어 설계

### Free (무료)

| 항목 | 내용 |
|------|------|
| 가격 | $0/월 |
| 레포 | 1개 (공개 레포만) |
| 실행 빈도 | 주 1회 |
| PR 생성 | 월 최대 4개 |
| 의사결정 로그 | 기본 (DECISIONS.md만) |
| 목적 | 제품 체험, 가치 입증, 바이럴 유도 |

**설계 의도**: 무료 사용자가 "에이전트가 실제로 유용한 PR을 만든다"는 것을 경험하게 한다. 공개 레포 제한으로 자연스럽게 제품이 알려지는 바이럴 효과도 기대.

### Pro ($29/월)

| 항목 | 내용 |
|------|------|
| 가격 | $29/월 (연간 결제 시 $24/월) |
| 레포 | 5개 (공개 + 비공개) |
| 실행 빈도 | 매일 |
| PR 생성 | 무제한 |
| 의사결정 로그 | 상세 (DECISIONS.md + 웹 대시보드) |
| 우선 지원 | 이메일 응답 48시간 내 |
| 목적 | 핵심 수익원, 1인 개발자 메인 플랜 |

**설계 의도**: 페르소나 "김지훈"의 지불 의향 범위($29-49/월) 하단에 설정. 넷플릭스 1개월 비용 수준으로 심리적 부담 최소화. 매일 실행 + 무제한 PR로 확실한 가치 차이를 느끼게 함.

**$29 가격 근거**:
- 타깃 사용자 페르소나의 지불 의향 하한선
- 경쟁 제품 대비: Copilot($10) < **Self-Improving Agent($29)** < Devin($500)
- 제공 가치 대비: 매일 1개 자율 PR = 주니어 개발자 시급 $25-40의 30분 작업과 유사

### Team ($79/월)

| 항목 | 내용 |
|------|------|
| 가격 | $79/월 (연간 결제 시 $66/월) |
| 레포 | 무제한 |
| 실행 빈도 | 매일 + 온디맨드 트리거 |
| PR 생성 | 무제한 |
| 의사결정 로그 | 상세 + Slack 알림 연동 |
| 우선 지원 | 이메일 응답 24시간 내 |
| 팀 기능 | 최대 5명 팀원 접근, 권한 관리 |
| 목적 | 2순위 타깃(스타트업 CTO) 수용 |

**설계 의도**: 1인 개발자가 팀으로 성장했을 때 자연스럽게 업그레이드. Devin($500/월)의 1/6 수준으로 소규모 팀에게 매력적인 대안 제시.

## 6. 무료 → 유료 전환 전략

1. **가치 체험 우선**: Free에서 첫 자율 PR을 받는 순간이 전환 핵심 트리거
2. **제한 체감**: 주 1회 실행 제한으로 "매일 받고 싶다" 욕구 유발
3. **비공개 레포 잠금**: 실제 사업 레포(비공개)는 Pro부터 → 진지한 사용자 전환 유도
4. **14일 Pro 체험**: 가입 시 14일간 Pro 기능 무료 제공, 체험 후 자연스러운 전환

## 7. 향후 과금 관련 과제

- Stripe 연동 구현 (Phase 2)
- 연간 결제 할인 구조 확정
- 초과 사용 시 요금 정책 (레포 수 초과 등)
- 환불 정책 수립
- 학생/오픈소스 할인 프로그램 검토

---

## 8. 결론

**SaaS 월정액 모델**을 채택하며, Free($0) / Pro($29) / Team($79) 3단계 티어로 운영한다. 핵심 수익원은 Pro 플랜이며, 타깃 사용자의 지불 의향($29-49/월) 하단에 가격을 설정하여 초기 전환율을 극대화한다.
