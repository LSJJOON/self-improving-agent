**1인 개발자를 선택한 이유:**
1. 시간이 가장 부족 — 에이전트의 "자율 PR" 가치를 가장 크게 체감
2. 혼자 결정 → 구매 전환 속도가 빠름
3. Devin($500/월) 대비 $29-49/월에 가장 민감하게 반응
4. IndieHackers, Twitter/X, ProductHunt 등 온라인 커뮤니티를 통한 마케팅 접근 용이

## 핵심 페르소나 카드

### 👤 김지훈 (Ji-hoon Kim)

| 항목 | 내용 |
|------|------|
| **나이** | 29세 |
| **직업** | 1인 SaaS 개발자 (풀타임 인디 해커) |
| **경력** | 대기업 3년 → 퇴사 후 독립 2년차 |
| **월 수입** | 250-450만원 (SaaS 구독 수익 + 간헐적 프리랜서) |
| **기술 스택** | TypeScript, React, Python, Supabase, Vercel |
| **주요 도구** | Notion, Linear, GitHub, Vercel, Stripe |
| **월 SaaS 지출** | 약 15-20만원 (Notion $16 + Linear $8 + Vercel $20 + 기타) |

#### 하루 일과
> "오전 9시 커피 한 잔과 함께 GitHub 이슈 확인. 코딩 4시간, 트위터 마케팅 1시간, CS 이메일 1시간. 저녁에 또 다른 아이디어가 떠오르는데 구현할 에너지가 없다."

#### 주요 고충 (Pain Points)

1. **시간 부족**: 아이디어는 넘치지만 실행할 시간이 없다. 기능 개발, 버그 수정, 마케팅, CS를 혼자서 다 처리한다.
2. **기술 부채 방치**: "나중에 리팩토링하자"는 다짐이 쌓인다. 리뷰해 줄 동료도 없다.
3. **번아웃**: 6개월째 같은 코드를 보고 있으면 작은 개선도 하기 싫다.
4. **일관성 부재**: 매일 조금씩 개선하고 싶지만, 실제로는 바쁜 날엔 아무것도 못 한다.
5. **고독감**: 코드 결정을 혼자 해야 한다. "이게 맞는 방향인지" 확인해 줄 사람이 없다.

#### 원하는 것 (Jobs to be Done)

> "내가 자는 동안에도 내 제품이 조금씩 나아지고 있으면 좋겠다."

- 매일 작은 개선이 자동으로 PR로 올라오길 원함
- 무엇을 왜 바꿨는지 설명이 있어야 신뢰할 수 있음
- 내 레포의 컨텍스트(백로그, 방향)를 이해하고 작업을 선택해 주길 원함
- 코드 품질보다 **제품 가치에 직결되는 작업**을 원함

#### 구매 결정 기준

| 기준 | 중요도 |
|------|--------|
| 실제로 유용한 PR을 만들어주는가 | ★★★★★ |
| 내 레포/제품을 이해하고 판단하는가 | ★★★★★ |
| 가격이 합리적인가 ($50/월 이하) | ★★★★☆ |
| 설정이 간단한가 (5분 내 온보딩) | ★★★★☆ |
| 내가 컨트롤할 수 있는가 | ★★★☆☆ |

#### 지불 의향

- **최대 지불 의향**: $49/월
- **편안한 가격대**: $19-29/월
- **비교 기준**: "Vercel Pro ($20/월)보다 더 유용하면 쓴다"
- **무료 티어 필요 여부**: 첫 달 무료 또는 레포 1개 무료로 먼저 경험 필요

## 타깃 세그먼트에서 제외한 이유

### 오픈소스 메인테이너
- PR 리뷰 부담은 크지만, **수익화에 비협조적** (무료/공개 문화)
- 자율 AI PR에 대한 커뮤니티 저항 가능성
- "내 스타일과 맞지 않는 PR이 올라오면 오히려 부담"

### 엔터프라이즈 개발팀
- 의사결정 속도 느림 (구매 사이클 3-6개월)
- 보안 검토 요구 (GitHub App 권한에 민감)
- 우리 초기 단계에서 대응 불가

## 마케팅 채널 추천

타깃 사용자 (1인 개발자)에게 효과적인 채널:

1. **IndieHackers** — 인디 해커 커뮤니티, 제품 리뷰/마일스톤 공유
2. **Twitter/X** — #buildinpublic 해시태그, 에이전트의 일일 PR 공개
3. **ProductHunt** — 런칭 시점에 집중적 트래픽 획득
4. **Hacker News (Show HN)** — 기술 친화적 얼리 어답터
5. **GitHub Trending** — 레포 자체를 제품 데모로 활용
